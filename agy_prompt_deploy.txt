## TAREA 1: Diagnosticar y arreglar application.properties

1. LEE el archivo actual:
   /mnt/c/Users/juanp/Documents/Proyectos/proyecto-final-web/web_project/web_project/java-backend/src/main/resources/application.properties

2. Revisa que tenga estas 3 variables separadas para la DB:
   - spring.datasource.url=${SPRING_DATASOURCE_URL}
   - spring.datasource.username=${SPRING_DATASOURCE_USERNAME}
   - spring.datasource.password=${SPRING_DATASOURCE_PASSWORD}
   
   Si la URL contiene credenciales incrustadas (user:pass@host), CORRIGELO para que solo sea la URL base.
   La URL correcta para Supabase es: jdbc:postgresql://HOST:PORT/DATABASE sin usuario ni password en la URL.

3. Verifica que todas las variables de entorno USADAS esten listadas en el archivo.
   Debe incluir:
   - SPRING_DATASOURCE_URL
   - SPRING_DATASOURCE_USERNAME
   - SPRING_DATASOURCE_PASSWORD
   - SPRING_MAIL_USERNAME
   - SPRING_MAIL_PASSWORD
   - PORT (default 8080)

4. Despues de modificar, ejecuta:
   cd /mnt/c/Users/juanp/Documents/Proyectos/proyecto-final-web/web_project/web_project/java-backend
   mvn compile -q
   Si falla, corrige y repite hasta que compile.

## TAREA 4: Configurar Render

Crea render.yaml en la raiz:
```yaml

  - type: web
    name: ml-lab-backend
    env: java
    plan: free
    buildCommand: cd web_project/web_project/java-backend && mvn clean package -DskipTests
    startCommand: java -jar web_project/web_project/java-backend/target/ml-lab-backend-1.0-SNAPSHOT.jar
```
